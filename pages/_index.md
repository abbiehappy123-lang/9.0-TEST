# 页面存档索引（pages/）

本目录存**具体页面的实现存档**（page 实例级），记录「这一个页面是怎么做的」：
用了哪些组件、哪些 token、版式如何、有哪些值得复用的做法与已知偏差。

> **page ≠ layout。**
> - `pages/` = 具体页面存档（实例级，一页一文件，如何做的记录）。
> - `layouts/` = 可复用版式骨架（抽象级，一类页一文件）。**当前未建**，
>   当同类页 **≥2 个** 时，再从这些 page 存档里提炼共性、另建 `layouts/`。
> - 本目录每条的「版式骨架 / 可复用点」两段，是将来抽 layout 的原料，
>   但在此之前它们只是 page 存档的一部分，不构成 layout。

## 单页存档格式（模板的模板）

新增 page 存档时，按以下结构写，保证格式统一、便于将来横向对比抽 layout：

```
# {页面名}

- 来源：Figma fileKey `xxx` node `xx:xx`
- 场景：一句话说明这是什么页、给谁用

## 版式骨架（自上而下）
逐层列出：层名 / 用什么组件或元素 / 关键位置尺寸
（← 将来抽 layout 时搬这段）

## 使用的组件
| 元素 | 库组件 + 变体 | 备注 |
（对齐 references/components.md，标明是实例还是手动拼装）

## 使用的 Token
只引用不复述值（值以 tokens.json / colors.md 为准）

## 资产
| 文件 | 用途 | 路径 |
（图片等，路径为 skill 内相对路径；上 git 后可换 raw URL 前缀）

## 可复用点 / 待组件化（TODO）
本页哪些做法值得沉淀、哪些元素应做成组件但尚未做
（← 将来抽 layout / 补组件的判断依据）

## 已知偏差
本页与规范红线不符之处，标明「实例偏差，不作规范依据」
```

## 升级为 layout 的时机

- 出现**第 2 个同类页**时，对比两份存档的「版式骨架 + 可复用点」。
- 共性稳定 → 提炼成 `layouts/{类型}.md`，各 page 存档改用 `[[link]]` 指向它。
- 红线永远只在 `references/custom-rules.md`，layout 与 page 均只引用不复述。
